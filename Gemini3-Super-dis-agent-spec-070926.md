1. 系統引言與架構概述
AURA-7 Hub v2 是一款專為「第三類（Class III）高風險醫療器材」流向追蹤、冷鏈監控、帳籍對帳及法規合規裁決所設計的自主代理作業系統（Agentic OS）。本系統採用全棧式（Full-Stack）架構，前端基於 React 18 與 Vite 構建，配合強大的動畫庫 motion/react（原 Framer Motion）實現平滑的介面轉換與微交互；後端採用輕量、高效的 Express 框架，並與 Google Gemini API 進行深度整合。
系統的核心設計理念是「自主代理（Agentic AI）」，將大語言模型（LLM）的推理能力從傳統的「問答對話」升級為「主動執行、多代理對辯、公式化數據合成及供應鏈衝擊預測」的智慧體。系統整體運行於高度安全的沙盒環境中，所有 API 密鑰與敏感數據均透過伺服器端（Server-Side）進行代理轉發，嚴防任何金鑰洩露至客戶端。

3. 後端架構與 API 設計
Express 後端負責充當安全代理（Security Proxy），隱藏所有訪問 Google Gemini 的敏感 API 密鑰。所有與 LLM 互動的邏輯，皆採用 Lazy Initialization（延遲初始化）模式，若環境變數中未配置 GEMINI_API_KEY，後端仍可安全啟動並向前端發送健康診斷，避免系統在加載時直接崩潰。

5. 本地個資去識別化與脫敏引擎 (Local Redaction Pipeline)
本系統將「隱私安全（Privacy-by-Design）」作為核心設計指導方針。在醫療器材合規稽核中，上傳的文件（如出庫單、醫院庫房簽收手札、臨床不良事件報告等）極易夾帶患者的敏感個資。若直接將這些數據送至雲端大模型，將面臨違反**台灣個人資料保護法（個資法）**與美國 HIPAA 的重大法律風險。

5.2 隔離執行緒安全保障
此脫敏引擎完全運行於用戶的瀏覽器沙盒執行緒中。任何包含原始敏感個資的文字，在「拖曳上傳」或「檔案載入」的瞬間，皆會立即完成正則過濾，並僅將**脫敏後的文本（Redacted Text）**寫入 React 的狀態變數 uploadedFiles.redactedContent。後續送交 gemini-3.1-flash-lite 進行法規判讀時，模型接觸到的僅有安全脫敏代碼。系統日誌亦會對此進行「[MASK/SECURE] 安全隔離」標註。
6. 對位矩陣（Comparison Matrix）平行處理機制
為了解決不同 LLM 在法規推理精準度、語意解析速度及營運成本間的平衡問題，AURA-7 Hub v2 設計了對位矩陣（Comparison Matrix）平行評估系統。
6.1 平行非同步請求架構
當使用者點擊「啟動矩陣對位評估」時，前端將同時向後端發射比較請求。後端 Express 伺服器會利用 JavaScript 的非同步併發能力，啟動對三個不同等級模型的調用：
gemini-3.1-flash-lite：定位為低時延、超高能效之初審代理。
gemini-3.5-flash：定位為兼顧高推理解析深度與中等時延之性能旗艦。
gemini-3.1-pro-preview：定位為高邏輯深度、能夠應對極複雜多層次法規條款之「深層推理代理」。
