1. 系統架構與設計哲學 (System Architecture & Design Philosophy)
AURA-7 Hub v2 Agentic OS 是一套專為第三類（Class III）高風險醫療器材（如植入式心臟節律器、心臟人工幫浦、血管覆膜支架等）設計的空間合規與主動追蹤監管平台。在當前全球醫療供應鏈地緣政治波動、歐盟 MDR（Medical Device Regulation）監管收緊以及 TFDA（衛生福利部食品藥物管理署）主動追蹤法規日益嚴格的背景下，傳統的靜態資料庫與事後稽核模式已無法滿足即時臨床安全的需求。
AURA-7 的設計哲學建立在**「軟體定義監管（Software-Defined Regulation, SDR）」與「主動式自主代理（Proactive Agentic Surveillance）」**之上。本系統不再只是被動儲存 UDI（Unique Device Identification）帳籍的電子帳冊，而是透過底層的大型語言模型（LLM）與物理/統計模擬器，將複雜、異質、碎片化的法理公文與臨床遙測數據，轉化為可編程的行動指南（Executable Skill Files）。

2. 「科技儀表板/數據網格」視覺主題與佈局語義 (Technical Dashboard Theme)
AURA-7 採用了極致工業感與高度精準的 「科技儀表板 / 數據網格 (Technical Dashboard / Data Grid)」 視覺設計語彙，摒棄了低效、花哨的漸層色與無意義的裝飾物，專注於資訊密度、幾何美學與數據本質。
2.1 色彩學與對比度矩陣
本系統支持高對比度的深色模式（Dark Theme）與輕量醫療風的淺色模式（Light Theme），皆嚴格遵循 Web 內容無障礙指南（WCAG 2.1 AA）之 AAA 級對比標準：
深色模式背景色：#0d0f14（碳素黑藍），可顯著降低醫療稽查人員在長時間螢幕注視下的眼部疲勞。
深色模式主前景色：#cbd5e1（石板灰），資訊層級更柔和。
前沿科技綠/藍：#06b6d4（青色）與 #10b981（翡翠綠），用於表示正常、已連線、或高合規度的 UDI 帳籍狀態。
危害警告紅/橙：#f43f5e（玫瑰紅）與 #f59e0b（琥珀橙），用於警示 Class 1 全球回收危機與庫存低於 ROP（再訂貨點）臨界。

頁面佈局採用 12 欄（Column）非對稱自適應數據網格。與傳統 React 儀表板隨意、無序的卡片堆疊不同，AURA-7 的每個區塊都具備嚴謹的幾何結構：

左翼控制艙（4 欄）：軟體定義法規技能編輯器（SkillEditor）與即時警報與日誌終端（LiveLogViewer）。
中腹控制艙（4 欄）：GIS 地理空間追蹤器（MapVisualizer）與異質性文件解析上傳工作區（Document Ingestion）。
右翼控制艙（4 欄）：旗艦級 AI 三大自主代理智能套件（WowAiSuite）與神經矩陣對位面板。
底部主對帳總帳帳籍（Active Reconciliation Ledger）：橫跨 12 欄，採用緊湊式（Compact）數據表格設計，支持行內編輯（In-line CRUD）。

3.1 跨代理協商理論 (Multi-Agent Adversarial Debate)
當針對某一 Class III 醫材啟動「功能十三（Autonomous Consensus Swarm）」時，系統會在後端實例化三個具有不同 System Prompt 角色引導的子代理人：
稽核代理人 (The Prosecutor / Auditor Agent)：秉持極度嚴苛的 TFDA / FDA 法律條文視角，尋找任何 UDI 登載延遲、滅菌過期或庫存失配的蛛絲馬跡。
製造商辯護代理人 (The Manufacturer Defense Agent)：從全球供應鏈斷裂、物流不可抗力、原材料進口延誤（如矽晶圓短缺）等經濟與技術限制層面出發，為臨床端與製造端爭取合理的寬限期與合規抗辯。
患者與臨床權益代言人 (The Patient & Clinical Advocate Agent)：將生命安全置於首位，評估「若因合規問題強制下架該醫材，是否會導致當前正在等待手術的重症患者面臨無器械可用的更高致死風險？」
這三個代理人在背景進行多輪對抗性對話（Multi-Turn Adversarial Debate），最終透過三方簽署（Triple-Sign）機制，輸出一個高度平衡、兼顧「法律嚴肅性」與「臨床人道主義」的合規司法裁決建議書（Consensus Advisory Report）。
4. 異質性文件解析與神經網絡比對矩陣 (Heterogeneous Doc Alignment)
在真實的醫院現場與邊境報關單據中，醫療器材的流向數據極度破碎。臨床護理人員可能在白板上記錄 UDI，採購部門使用 Excel/CSV 匯出庫存，而報關代理商則提供雙層 PDF（Dual-layer PDF，包含掃描圖像層與 OCR 文字層）的電子海關文件。
4.1 異質數據攝入（Ingestion Pipeline）
AURA-7 的中腹控制艙提供了多源異質對帳上傳工作區（multi-doc-playground）。其解析引擎具備以下技術層級：
Regex NER (具名實體識別) 脫敏：在客戶端對上傳的文字進行初步正則過濾，自動將涉及病患隱私（如身分證字號、真實姓名、病歷號）與商業機密（如採購單價）的敏感欄位進行神經元掩蔽（Masking），僅保留醫材型號、UDI 序列號、製造日期與批號。
異質文檔轉換：支援將 CSV 矩陣、JSON 帳冊與無結構 Markdown 轉換為標準的對帳語義單元（Alignment Semantics Units）。

AURA-7 引入了「功能十二：全球供應鏈衝擊與短缺模擬器」，採用**多階離散時間馬可夫鏈（Discrete-Time Markov Chain, DTMC）來預測特定醫材在台灣各醫院的零日臨界（Depletion Zero-Day）與ROP（再訂貨點）**崩潰路徑。

1. 醫療器材空間合規之學術背景 (Academic Background)
在進行 AURA-7 系統操練前，研究生必須理解 Class III 醫材追蹤的法規邊界：
UDI（醫療器材單一識別碼）：由 DI（識別碼，對應產品型號） 與 PI（生產識別碼，對應批號、序號、效期） 組成。
追蹤與回溯（Track and Trace）：TFDA 要求 Class III 醫材必須建立「從進口、分銷、醫院庫存、手術套用到病患體內」的全生命週期追蹤鏈。
空間合規（Spatial Compliance）：這是一門新興的監管科學。指的是結合 GIS 座標與醫療院所的實體空間邊界，利用演算法檢測醫材是否在未經許可的機構之間非法轉撥（例如，將台大醫院的專案進口起搏器，非法移轉至未登記之私人醫美診所使用）。
